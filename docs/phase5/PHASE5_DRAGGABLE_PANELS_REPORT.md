# Phase5 悬浮文字框 UX 修复报告

## 处理内容

1. “对话文字”和“文字输入”两张卡片分别拥有独立拖动位置。
2. 拖动区域只放在卡片标题栏，避免输入框编辑与拖动手势冲突。
3. 对话文字卡最大高度限制为 210dp。
4. 用户 ASR 最多显示 2 行，助手回复最多显示 4 行，超出显示省略号。
5. 卡片宽度限制在适合手机屏幕的范围内。
6. 拖动范围限制在屏幕附近，降低卡片完全丢出屏幕的概率。
7. 不修改语音、连接、MCP、KWS、设置开关和默认值。

## 交互

按住卡片顶部的“对话文字 / 文字输入”标题栏并拖动即可移动。
输入框本体仍用于输入文字，不会触发拖动。
