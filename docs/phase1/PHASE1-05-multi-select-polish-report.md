# Phase1-05 多选与体验打磨交付报告

## 本阶段目标

把 Phase 1 的基础体验补齐，收尾纯手动便签 MVP。重点是长按多选、批量操作、空状态与 Snackbar 反馈。

## 本阶段完成内容

- 列表长按便签进入多选模式。
- 多选模式下点击卡片切换选中状态。
- 支持全选 / 全不选 / 退出多选。
- 普通列表、待办、已完成、置顶、tag 筛选、搜索结果中均可多选。
- 普通/活跃便签支持批量置顶或取消置顶。
- 当选中项全部是待办便签时，支持批量已完成或取消完成。
- 活跃便签支持批量软删除。
- 活跃便签支持批量加 tag。
- 最近删除中支持批量恢复和批量彻底删除。
- 批量删除和彻底删除增加确认弹窗。
- 主要批量操作增加 Snackbar 反馈。
- 多选时隐藏新建与语音悬浮入口，避免误触。

## UI 变更

- 便签卡片支持长按进入多选。
- 多选时卡片左侧显示选择圆点。
- 选中卡片显示蓝色描边。
- 多选工具条以紧凑横向按钮呈现。
- 删除、彻底删除使用红色危险按钮。
- 批量结果通过 Snackbar 显示。

## 验收方式

1. 长按任意便签进入多选。
2. 点击其他便签可追加或取消选择。
3. 点击全选后当前可见结果全部选中。
4. 在普通列表中批量置顶，列表状态应更新。
5. 在置顶筛选中选中全部已置顶便签，应显示“取消置顶”。
6. 在待办筛选中选中待办，可批量标记完成。
7. 在已完成筛选中选中待办，可批量取消完成。
8. 批量删除后便签从活跃列表消失并进入最近删除。
9. 最近删除中可批量恢复。
10. 最近删除中可批量彻底删除，且需二次确认。
11. 批量加标签后，便签卡片显示新标签，并可通过 tag 抽屉筛选。

## 已知限制

- 批量操作目前逐条调用已有 UseCase，未额外新增批量 Repository API。
- 多选加标签为追加标签，不提供批量移除标签。
- 批量完成只在选中项全部为待办便签时显示。
