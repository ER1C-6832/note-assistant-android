# Phase1-05 收尾修正：详情归档、拼音搜索、双列列表与深色背景

## 背景

本次继续收尾 Phase1，不推进 Phase2。目标是在不改变既有主体功能的前提下，补齐详情页归档入口、提升首页列表密度、增强搜索容错，并完善主题颜色选项。

## 本次改动

1. 便签详情页新增归档能力
   - 普通/待办便签详情页新增“归档”按钮。
   - 已归档便签详情页显示“取消归档”。
   - 操作仍通过 `SetNoteArchivedUseCase`，不从 UI 直接访问 DAO。

2. 首页列表更紧凑
   - 便签正文只显示一行，超过部分用省略号。
   - 卡片内边距和行距收紧。
   - 标签 chip 最多显示两个，减少卡片高度。

3. 首页新增单列/双列切换
   - 在搜索统计行右侧增加“单列/双列”切换。
   - 默认仍为单列。
   - 双列模式复用同一批 `NoteCard`，保留点击、长按多选、checkbox 等现有交互。

4. 搜索增强
   - 搜索仍保持每输入/删除一个字符即时刷新。
   - 支持标题、正文、标签的原文匹配。
   - 增加轻量拼音/首字母匹配，例如“王总”可用 `wangzong`、`wangz`、`wz` 搜到。
   - 数字/英文继续按子串匹配，例如 `114514` 可用 `11` 搜到。

5. 主题颜色补充黑色
   - 主界面背景新增“深色”和“黑色”。
   - Tag 列表背景新增“深色”和“黑色”。
   - 首页标题和 Tag 抽屉文字会根据背景明暗做基本适配。

6. 业务层检查
   - 本次保持 UI -> ViewModel -> UseCase -> Repository/DAO 的路径。
   - UI 层没有新增 DAO 直连。
   - 多选、归档、删除、置顶、完成、标签修改继续由 UseCase 入口承接，便于后续接语音命令。

## 验收建议

- 在详情页打开普通便签，点击“归档”，确认便签从首页消失并进入已归档。
- 在已归档打开便签，点击“取消归档”，确认回到普通列表/标签筛选范围。
- 搜索 `wangzong`、`wangz`、`wz`，确认能匹配“王总”。
- 搜索数字便签内容时确认子串匹配正常。
- 在首页切换双列，确认长按多选、checkbox、进入详情都不受影响。
- 设置主界面/Tag 列表背景为黑色，确认主要文字可读。
